Spark(string element, element context).event(string type, function callback)

Parameters

  • element - The CSS selector string to your element.
  • context - If supplied, the context in which you wish the selection to take place.
  • type - The type of event that has to take place in order for it to be triggered. Possible types are as follows.
    • load
    • unload
    • blur
    • change
    • focus
    • reset
    • select
    • submit
    • abort
    • keydown
    • keypress
    • keyup
    • click
    • dblclick
    • mousedown
    • mousemove
    • mouseout
    • mouseover
    • mouseup
  • callback - The function you wish to be run when the element is clicked, passes the event to the callback function.

Returns

Passes the event's return to the callback. For events such as click you can get the clicks location with pageX and pageY. This is usually not cross browser but Spark fixes this on runtime so you can use one value for all browsers.

The same fixes apply to the target / srcElement differences in some browsers. You just need to use event.target because Spark fixes these differences.

Examples

Set a click event with a callback function.

Spark('a.myButton').event('click', function(e) {
    alert('pageX = ' + e.pageX);
});

Set a click event on a link and prevent it from changing the page by returning false.

Spark('a.myButton').event('click', function(e) {
    console.log('This browser is not going anywhere!');
    return false;
});
